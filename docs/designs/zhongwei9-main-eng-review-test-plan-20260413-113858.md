## Affected Modules
- doc2know/splitter.py — 核心三阶段拆分逻辑
- doc2know/generator.py — 版本目录管理扩展
- doc2know/utils.py — JSON 解析工具函数
- doc2know/cli.py — cleanup 命令

## Key Interactions to Verify
1. **正常拆分流程**：输入 50+ 页文档 → 生成 3-5 个分类文件
2. **版本目录创建**：时间戳格式正确，latest 软链接更新
3. **降级策略**：
   - 阶段 1 失败 → 单文件输出
   - 阶段 2 失败 → 使用原始主题继续
   - 阶段 3 失败 → 跳过失败分类
4. **并发控制**：阶段 3 最多 3 个并发 LLM 调用

## Edge Cases
- 空文档/内容过少
- LLM 返回无效 JSON
- Token 超限
- 磁盘权限不足（版本目录创建失败）
- 同名分类冲突

## Critical Paths
- End-to-end: Word/PDF → 3-5 Markdown files in versioned directory
- Error recovery: Any stage failure → graceful degradation

## LLM Eval Tests Required
- Stage 1 prompt: extract_topics.txt
- Stage 2 prompt: merge_categories.txt (conservative & aggressive)
- Stage 3 prompt: extract_content.txt

## Test Data
- Python 官方教程 PDF (~200 pages)
