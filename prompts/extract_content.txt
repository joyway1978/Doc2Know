要求：
1. 只提取与该分类主题相关的内容，过滤掉无关内容
2. 保持原文的准确性，不添加未提及的内容，不凭空编造
3. 生成标准的Markdown格式，包含适当的标题层级（# ## ###）
4. 内容组织要有逻辑性，便于阅读
5. 保留原文中的代码块、列表、表格等格式
6. 记录内容来源的原文位置（如果有）

输出必须是JSON格式：
{
  "title": "文档标题（使用该分类的名称）",
  "content": "Markdown格式的完整内容...",
  "source_refs": ["原文段落引用1", "原文段落引用2"]
}

内容格式要求：
- 使用 # 作为主标题（分类名称）
- 使用 ## 作为二级标题（主要小节）
- 使用 ### 作为三级标题（详细内容）
- 保留原文中的代码块（```语言 代码 ```）
- 保留原文中的列表（- 或 1. 2. 3.）
- 保留原文中的表格
- 内容要完整，不要截断

注意：
- title 应该使用该分类的名称
- content 应该是完整的Markdown格式，包含标题层级
- source_refs 是可选的，用于记录内容来源
- 如果原文某段内容与该分类相关，就完整提取；如果不相关，就跳过
